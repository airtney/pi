# 迷你我的世界（Mini Minecraft）

用 Three.js 实现的第一人称体素方块沙盒，向《我的世界》（Minecraft）致敬的迷你复刻。
纯前端单页，零构建、零安装、零图片资源（贴图全部程序化生成）。

## 怎么玩

直接用浏览器打开 `index.html` 即可（需联网加载 Three.js CDN）。
如果浏览器对 `file://` 有限制，也可以起一个本地静态服务器：

```bash
cd fable
python3 -m http.server 8000
# 打开 http://localhost:8000
```

点击「点击开始」锁定鼠标进入游戏。

### 操作

| 按键 | 功能 |
| --- | --- |
| `W` `A` `S` `D` / 方向键 | 移动 |
| 鼠标 | 视角（PointerLock） |
| `空格` | 跳跃（水中为上浮） |
| `Shift` | 疾跑 |
| 鼠标左键 | 破坏准星对准的方块 |
| 鼠标右键 | 在对准面上放置方块 |
| `1` `2` `3` `4` / 滚轮 | 切换热键栏方块：草方块 / 泥土 / 石头 / 木头 |
| `Esc` | 解锁鼠标（暂停） |

## 功能清单

- **第一人称控制**：PointerLockControls + WASD，跳跃、疾跑
- **体素世界**：128×48×128，含草方块、泥土、石头、木头、树叶、沙子、水、基岩
- **程序化地形**：多倍频值噪声生成山丘、平原、湖泊与沙滩，底层为不可破坏的基岩
- **树木**：草地上随机生成带树冠的橡树
- **破坏 / 放置**：体素 DDA 射线拾取，目标方块黑色线框高亮；破坏有方块碎裂粒子与音效（WebAudio 合成，无音频文件），放置会做玩家碰撞检查，防止把方块放进自己身体
- **热键栏**：1-4 数字键或滚轮切换，带像素图标与选中高亮
- **物理**：重力、跳跃、AABB 逐轴扫掠碰撞（分子步防高速穿墙）、世界边界约束，不会穿墙或掉出世界；水中有浮力与阻力，可游泳，水下有蓝色滤镜
- **渲染**：按 16×16 分块合并网格，只生成暴露面；16×16 像素程序化贴图图集（NearestFilter 保持像素风）；天空、雾、平行光 + 环境光 + 半球光的白天光照、面向亮度差异（顶亮底暗）、飘动的方块云
- **HUD**：十字准星、坐标、FPS、当前方块名称

## 文件结构

| 文件 | 说明 |
| --- | --- |
| `index.html` | 页面入口，import map 从 CDN 加载 Three.js r169 后启动游戏 |
| `style.css` | 开始菜单、准星、热键栏、HUD 等样式 |
| `textures.js` | 方块定义 + Canvas 程序化像素贴图图集 |
| `world.js` | 值噪声地形生成、树、水、分块网格构建、体素射线 |
| `player.js` | 玩家物理：重力、AABB 碰撞、游泳、出生点选择 |
| `app.js` | 主程序：场景、输入、破坏/放置、热键栏、粒子、音效、主循环 |

## 技术说明

- Three.js **r169**（满足 r160+ 要求），通过 jsDelivr CDN 以 ES Module 加载；游戏逻辑为普通脚本，挂在 `window.MC` 命名空间下，因此 `file://` 直开也能运行。
- 方块拾取使用 Amanatides & Woo 体素遍历（DDA），比对网格做三角形射线检测更快更稳。
- 修改方块后只重建所在分块（及跨界时的相邻分块）的网格。
- 无任何后端、无 npm install、无构建步骤。
